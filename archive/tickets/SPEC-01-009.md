# SPEC-01-009: Integration test — `world-index build animalia` verifies counts, determinism, drift, and incremental sync

**Status**: COMPLETED
**Priority**: HIGH
**Effort**: Medium
**Engine Changes**: Yes — adds `tools/world-index/tests/integration/build-animalia.test.ts` and lands the required parser/determinism fixes in `tools/world-index/src/parse/yaml.ts` and `tools/world-index/src/commands/shared.ts` so the real animalia corpus can pass the end-to-end verification gate truthfully.
**Deps**: SPEC-01-008 (CLI + all commands)

## Problem

SPEC-01 §Verification requires three end-to-end checks on the real animalia world: (a) `world-index build animalia` succeeds with node counts matching the live empirical scale (47 CF, 18 CH, 20 MR, 17 PA, 2 characters, 2 DAs, 12 PRs + 3 BATCH manifests, 7 NCPs + 1 NCB manifest, 1 AU, 1 RP); (b) re-running produces identical `content_hash` values (determinism); (c) `world-index sync` after a file edit reparses only that file (incremental selectivity); plus (d) `world-index verify` flags drift after a manual edit. Without this ticket, the prior eight tickets could each pass their own unit tests while the composition at scale fails silently. This is the structural SPEC-01 completion gate that SPEC-08 Phase 1 depends on.

## Assumption Reassessment (2026-04-22)

1. Empirical animalia counts verified against the live tree on 2026-04-22: 47 `canon_fact_record`, 18 `change_log_entry`, 20 `mystery_reserve_entry`, 17 `adjudication_record`, 2 `character_record`, 2 `diegetic_artifact_record`, 12 `proposal_card`, 3 `proposal_batch`, 7 `character_proposal_card`, 1 `character_proposal_batch`, 1 `audit_record`, and 1 `retcon_proposal_card`. Counts are not frozen; the test re-enumerates from source at runtime instead of hardcoding them.
2. Initial reassessment exposed two same-seam blockers in the live `world-index` implementation: `build animalia` exited `4` because canonical-ledger YAML contained prose list items that the parser rejected as syntax errors, and synthetic IDs for fallback YAML nodes were path-dependent because `parseWorldFile` passed an absolute path into `extractYamlNodes`. This ticket owns both fixes because the end-to-end proof was not truthful without them.
3. Test strategy: run the integration against a temp copy of `worlds/animalia/` so the suite never mutates the real world tree and can edit files safely during `sync`/`verify` checks. Copy is via `fs.cpSync({ recursive: true })`.
4. Cross-artifact boundary: this test runs the real CLI entry point against the real animalia world, producing a real `worlds/animalia/_index/world.db` under the temp root. It verifies the complete SPEC-01 producer pipeline end-to-end; SPEC-02's MCP layer remains out of scope because it is a reader.
5. FOUNDATIONS principle under audit: **Rule 6 (No Silent Retcons)**. The drift-detection sub-test is the structural proof that SPEC-01 preserves Rule 6 under post-build edits.
6. Performance target: SPEC-01 §Risks sets the aspirational build time at <2s on commodity hardware. The honest pass/fail gate for this ticket remains `< 30s` to avoid CI/load flake.

## Architecture Check

1. One integration test file rather than multiple sibling files keeps the temp-copy setup and DB helpers centralized while still expressing each invariant as its own test.
2. Using `fs.cpSync` on a temp directory means tests run in parallel safely (each test session gets its own temp root). No mutation of the real world state.
3. The required-consequence parser change is narrow: recover only the malformed ledger list-item scalars that the real corpus already uses, keep the raw node body untouched, and preserve the existing hard failure path for genuinely unrecoverable YAML.
4. Determinism now compares two fresh builds under different temp roots after removing absolute-path leakage from fallback YAML node IDs. Alternative — weakening the test to ignore node IDs — was rejected because the node identity contract itself needed to be deterministic.

## Verification Layers

1. **Count invariant**: `SELECT node_type, COUNT(*) FROM nodes GROUP BY node_type` matches the expected counts (re-enumerated from the animalia fixture at test start; SPEC-01 §Verification mandate).
2. **Determinism invariant**: `content_hash` over every `node_id` is equal across two independent builds → re-runnable without drift.
3. **Incremental selectivity invariant**: touching one file's byte contents causes `sync` to reparse only that file — measured via `file_versions.last_indexed_at` on every file before and after.
4. **Drift detection invariant**: editing one line of a source file and running `verify` produces a `drift_check` fail row keyed to the edited node.
5. **Schema stability invariant**: the `index_version.txt` written by `build` equals `CURRENT_INDEX_VERSION` exactly (migration baseline is frozen at 1 per SPEC-01-002).
6. **File-enumeration coverage invariant**: every `.md` file under `worlds/animalia/` that matches the include-list appears in `file_versions`; every file in the exclude-list does NOT (grep-proof on `file_versions`).

## What to Change

### 1. `tools/world-index/tests/integration/build-animalia.test.ts`

Structure:

- `beforeEach`: `fs.cpSync('worlds/animalia', tmpdir + '/worlds/animalia', { recursive: true })`. Re-enumerate counts from the fresh copy (don't hardcode — SPEC-01-009's spec comment says counts grow over time; the test computes expected counts from the filesystem at test start).
- `test('build succeeds + node counts match empirical scale')`: invoke `build(tmpdir, 'animalia')`. Assert exit code 0. Open the DB, run count queries, compare to the pre-computed expected counts. Must cover canon_fact_record (47), change_log_entry (18), mystery_reserve_entry (20), adjudication_record (17), character_record (2), diegetic_artifact_record (2), proposal_card (12), proposal_batch (3), character_proposal_card (7), character_proposal_batch (1), audit_record (1), retcon_proposal_card (1), plus non-structural counts for section/subsection/invariant/ontology_category/named_entity.
- `test('build is deterministic across fresh runs')`: `fs.cpSync` into two temp roots. Build each. `SELECT node_id, content_hash FROM nodes ORDER BY node_id` must be identical. Any mismatch fails the test with the diverging node id.
- `test('sync reparses only touched files')`: `build` once. Snapshot `file_versions.last_indexed_at` per file. Modify the byte content of one domain file (e.g., append a space to `ECONOMY_AND_RESOURCES.md`). `sync`. Assert `last_indexed_at` for that file changed and no other file's `last_indexed_at` changed.
- `test('verify flags drift after manual edit')`: `build`. Overwrite one line of a temp-copied indexed source file (the landed test edits `adjudications/PA-0001-accept_with_required_updates.md`). `verify` must return 1 and the `validation_results` table must contain a `drift_check` fail row naming the edited node.
- `test('build completes within 30s')`: measure wall-clock time. Assert `< 30_000` ms. Cross-platform note: CI variance is acceptable; if we hit flake, move to SPEC-01 §Risks streaming-parse mitigation in a follow-up.
- `test('all indexable files appear in file_versions')`: enumerate the animalia tree, assert every non-excluded path has a row in `file_versions`.

### 2. `tools/world-index/src/parse/yaml.ts`

Add a narrow recovery pass for ledger-style prose list items that currently parse as YAML syntax errors, so the real animalia ledger can build without weakening the parser's behavior on unrecoverable YAML.

### 3. `tools/world-index/src/commands/shared.ts`

Pass relative file paths into `extractYamlNodes` so fallback synthetic IDs remain stable across temp roots and fresh builds.

### 4. `tools/world-index/tests/yaml.test.ts`

Add a targeted regression proving the YAML recovery path keeps malformed ledger prose in the YAML-backed path instead of downgrading it to prose hashing.
