# SPEC-01-009: Integration test — `world-index build animalia` verifies counts, determinism, drift, and incremental sync

**Status**: PENDING
**Priority**: HIGH

## Problem

SPEC-01 §Verification requires three end-to-end checks on the real animalia world: (a) `world-index build animalia` succeeds with node counts matching the empirical-scale numbers from the spec (47 CF, 18 CH, 20 MR, 17 PA, 2 characters, 2 DAs, 12 PRs + 3 BATCH manifests, 7 NCPs + 1 NCB manifest, 1 AU, 0 RPs); (b) re-running produces identical `content_hash` values (determinism); (c) `world-index sync` after a file edit reparses only that file (incremental selectivity); plus (d) `world-index verify` flags drift after a manual edit. Without this ticket, the prior eight tickets could each pass their own unit tests while the composition at scale fails silently. This is the structural SPEC-01 completion gate that SPEC-08 Phase 1 depends on.

## Architecture Check

1. One integration test file rather than four smaller ones keeps the setup/teardown (copying animalia, opening the DB, cleanup) declared once. Each assertion sub-block is independent; a failure in determinism doesn't obscure a count failure. Alternative — four sibling files — was rejected for copy-animalia duplication.
2. Using `fs.cpSync` on a temp directory means tests run in parallel safely (each test session gets its own temp root). No mutation of the real world state.
3. Determinism test runs two builds under different temp roots (same source) and compares the resulting `content_hash` values across all nodes. Alternative — run twice in the same DB — would conflate "deterministic parser" with "idempotent CRUD"; separate builds exercise both parts correctly.

## Verification Layers

1. **Count invariant**: `SELECT node_type, COUNT(*) FROM nodes GROUP BY node_type` matches the expected counts (re-enumerated from the animalia fixture at test start; SPEC-01 §Verification mandate).
2. **Determinism invariant**: `content_hash` over every `node_id` is equal across two independent builds → re-runnable without drift.
3. **Incremental selectivity invariant**: touching one file's byte contents causes `sync` to reparse only that file — measured via `file_versions.last_indexed_at` on every file before and after.
4. **Drift detection invariant**: editing one line of a source file and running `verify` produces a `drift_check` fail row keyed to the edited node.
5. **Schema stability invariant**: the `index_version.txt` written by `build` equals `CURRENT_INDEX_VERSION` exactly (migration baseline is frozen at 1 per SPEC-01-002).
6. **File-enumeration coverage invariant**: every `.md` file under `worlds/animalia/` that matches the include-list appears in `file_versions`; every file in the exclude-list does NOT (grep-proof on `file_versions`).
