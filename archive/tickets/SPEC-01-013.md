# SPEC-01-013: Use canonical frontmatter IDs for file-backed record nodes

**Status**: PENDING
**Priority**: HIGH
**Effort**: Medium
**Engine Changes**: Yes — `tools/world-index/src/parse/prose.ts` needs frontmatter-aware node-id selection for whole-file records, with corresponding test and integration updates.
**Deps**: `archive/tickets/SPEC-01-006.md`, `archive/tickets/SPEC-01-008.md`, `docs/FOUNDATIONS.md`, `specs/SPEC-04-validator-framework.md`

## Problem

Whole-file records such as diegetic artifacts, characters, proposals, and audits currently index under slug/path-derived node ids instead of their declared frontmatter ids. In live animalia data, `worlds/animalia/diegetic-artifacts/a-season-on-the-circuit.md` declares `artifact_id: DA-0001`, but the indexed node id is `animalia:a-season-on-the-circuit.md:a-season-on-the-circuit:0`. As a result, `derived_from: [DA-0001]`, `source_artifact_id: DA-0001`, and similar cross-file references cannot resolve through `cross_file_reference`.

## Assumption Reassessment (2026-04-22)

1. `tools/world-index/src/parse/prose.ts` routes subdirectory markdown such as `diegetic-artifacts/*.md`, `characters/*.md`, `proposals/*.md`, `character-proposals/*.md`, and `audits/*.md` through `createWholeFileRecord()`, which currently constructs the node id from file path + heading path and never inspects frontmatter identifier fields.
2. Live animalia files carry canonical ids in frontmatter: `artifact_id: DA-0001` / `DA-0002`, `character_id: CHAR-0001` / `CHAR-0002`, `proposal_id: PR-0010` and `NCP-0001`, and `audit_id: AU-0001`.
3. Live DB evidence shows the only currently unresolved `derived_from` target is `DA-0001` (`3` edges), and the indexed `diegetic_artifact_record` nodes use slug-derived ids rather than `DA-*`.
4. `specs/SPEC-04-validator-framework.md` requires every id referenced in `derived_from`, `required_world_updates`, `affected_fact_ids`, and related fields to resolve to an indexed node. The current whole-file node-id contract violates that requirement for frontmatter-backed record types.
5. Cross-artifact boundary under audit: frontmatter schema for file-backed records, `NodeRow.node_id`, and all graph consumers that expect structured ids (`inspect`, retrieval, validators, proposal/adjudication references).
6. Mismatch + correction: this should be solved generically for file-backed record classes with canonical frontmatter ids, not patched narrowly for `DA-*` only.

## Architecture Check

1. The clean solution is to parse just enough frontmatter to select the canonical node id for whole-file record types that already declare one, while leaving body hashing and node typing otherwise unchanged. That preserves the authored identity instead of manufacturing a second one.
2. No backwards-compatibility aliasing should create duplicate rows under both slug ids and canonical ids. The canonical frontmatter id should become the single node id when present.

## Verification Layers

1. A diegetic artifact with `artifact_id: DA-0001` indexes as `node_id='DA-0001'`. -> targeted parser test
2. A character with `character_id: CHAR-0001`, a proposal with `proposal_id: PR-0010`, a character proposal with `proposal_id: NCP-0001`, and an audit with `audit_id: AU-0001` each index under their declared ids. -> targeted parser test
3. Animalia `DA-0001` references resolve after build. -> integration test + readonly DB query
4. A whole-file record with missing or malformed canonical id falls back to the existing deterministic synthetic/slug path rather than inventing a bad structured id. -> edge-case parser test

## What to Change

### 1. Add frontmatter-aware id selection for whole-file records

Teach `createWholeFileRecord()` to parse the opening YAML frontmatter and prefer the canonical id field for node types that define one: `artifact_id`, `character_id`, `proposal_id`, and `audit_id`.

### 2. Keep the contract generic across file-backed record families

The same mechanism should cover `diegetic-artifacts`, `characters`, `proposals`, `character-proposals`, and `audits`, with explicit tests for each family that currently carries a canonical id field.

### 3. Add live graph regression coverage

Prove that the animalia `DA-0001` seam is fixed and that the new id contract does not create duplicate whole-file nodes.
