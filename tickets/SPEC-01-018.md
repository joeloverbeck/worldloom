# SPEC-01-018: Retune heuristic entity extraction against operational-label false positives

**Status**: PENDING
**Priority**: HIGH
**Effort**: Medium
**Engine Changes**: Yes — `tools/world-index` heuristic entity extraction, stoplist/filtering logic, and live regression coverage.
**Deps**: `archive/specs/SPEC-01-world-index.md`, `archive/tickets/SPEC-01-010-entity-heading-seam.md`, `tickets/SPEC-01-017.md`

## Problem

The production `animalia` index is structurally valid but semantically noisy on the `named_entity` surface. Readonly audit queries on 2026-04-22 showed `614` `named_entity` rows, `608` of which had `Kind: unknown`, with obvious non-entity examples such as `Adds Mystery Reserve`, `All Phase`, `Age Details`, `Action Report`, and `Access Path`. These are being persisted as first-class entity nodes and `mentions_entity` edges, which pollutes the graph that downstream retrieval and audit tooling will consume.

## Assumption Reassessment (2026-04-22)

1. The live heuristic in `tools/world-index/src/parse/entities.ts` still scans broad prose-node bodies with `CAPITALIZED_MULTIWORD_REGEX` after the heading/body seam fix from `archive/tickets/SPEC-01-010-entity-heading-seam.md`.
2. Sample readonly DB probes on 2026-04-22 showed false-positive entity mentions coming from operational labels and artifact titles rather than world entities: `Adds Mystery Reserve` was emitted from the `CANON_LEDGER.md` change-log section body, `All Phase` from adjudication discovery metadata, and `Action Report` from a diegetic artifact title.
3. `archive/specs/SPEC-01-world-index.md` positions `named_entity` nodes as a machine-facing semantic layer backed by `entity_mentions`. Persisting workflow labels and structural phrases as entities weakens that contract.
4. Shared boundary under audit: heuristic candidate collection in `parse/entities.ts`, `entity_mentions`, virtual `named_entity` rows, and the retrieval/validator consumers that will query those surfaces directly from SQLite.
5. FOUNDATIONS principle under audit: the index should reinforce the world's constrained model, not turn operational labels and section furniture into pseudo-canon entities.
6. Adjacent issue classification: `tickets/SPEC-01-017.md` owns provenance normalization for synthetic entity rows. This ticket owns which entities should exist at all.

## Architecture Check

1. The clean approach is to make the heuristic more conservative at candidate-generation time and/or candidate-filtering time, rather than teaching downstream consumers to subtract a growing bag of obvious false positives.
2. No backwards-compatibility alias path should keep emitting the noisy entities alongside a cleaned set.

## Verification Layers

1. Known false positives (`Adds Mystery Reserve`, `All Phase`, `Age Details`, `Action Report`, `Access Path`) are absent from the rebuilt `named_entity` surface. -> readonly DB query
2. Legitimate entities already covered by registry-backed or strong heuristic evidence remain present after the retune. -> targeted tool command + readonly DB query
3. The overall heuristic output becomes materially less "unknown operational label" heavy on the live `animalia` build. -> readonly DB query/manual review

## What to Change

### 1. Retune heuristic candidate extraction

Tighten `tools/world-index/src/parse/entities.ts` so operational labels, workflow metadata phrases, and structural titles stop becoming persisted entities in common worldloom files.

### 2. Add live regression examples

Encode the audited false positives in fixture/unit coverage and, where appropriate, a live `animalia` integration assertion so the noisy cases cannot regress silently.

### 3. Preserve real entity recall

Prove that the retune still keeps legitimate multiword entities that matter to world retrieval instead of solving the problem by effectively disabling heuristic extraction.

## Files to Touch

## Acceptance Criteria

### Tests That Must Pass

1. A rebuilt `animalia` index contains `0` `named_entity` rows for `Adds Mystery Reserve`, `All Phase`, `Age Details`, `Action Report`, and `Access Path`.
2. Existing and new entity extraction tests still prove legitimate entities are emitted where expected.
3. The live `animalia` build shows a materially reduced false-positive `Kind: unknown` surface without suppressing registry-backed entities.

### Invariants

1. `named_entity` rows should represent world entities or deliberate retrieval targets, not workflow labels or structural phrases.
2. The heuristic path remains conservative and additive; downstream consumers should not need a second noise-filtering layer to recover the intended entity graph.
