# SPEC-01-004: Pass 1 markdown AST + Pass 4 canonical-form hashing

**Status**: PENDING
**Priority**: HIGH
**Effort**: Medium
**Engine Changes**: Yes — adds `tools/world-index/src/parse/markdown.ts` (Pass 1), `tools/world-index/src/parse/canonical.ts` (Pass 4), and `tools/world-index/src/hash/content.ts`. Introduces the deterministic-hashing discipline every downstream parser and verifier relies on.
**Deps**: SPEC-01-001 (package scaffold), SPEC-01-003 (types)

## Problem

SPEC-01's three determinism guarantees — "re-run produces identical `content_hash` values across all nodes", "anchor_checksum detects surrounding-context drift", "`world-index verify` flags post-edit drift" — all rest on two modules: a cross-platform-stable markdown parser (Pass 1) and a deterministic canonical-form + hashing module (Pass 4). Without these, the patch engine (SPEC-03) cannot safely compare `expected_content_hash` values and the drift detector is meaningless. Pass 1 produces the mdast consumed by Passes 2 and 3; Pass 4 closes the pipeline with the two hash columns the schema already reserves (`nodes.content_hash`, `nodes.anchor_checksum`).

## Assumption Reassessment (2026-04-22)

1. SPEC-01 §Parser pipeline Pass 1 mandates `remark-parse` with position tracking; extracted node categories are `heading`, `list`, `listItem`, `paragraph`, `html` (for comments), `code` (for fenced yaml). SPEC-01-001's `package.json` pins `unified`, `remark-parse`, `remark-gfm` — `remark-gfm` is required because worldloom domain files carry GFM tables carrying attribution comments in cells (per reassessment evidence: 96 attribution comments in INSTITUTIONS.md tables).
2. SPEC-01 §Parser pipeline Pass 4: for YAML nodes, `content_hash = sha256(serialize_stable(parsed))` (keys sorted, whitespace normalized); for prose, `content_hash = sha256(normalize_whitespace(body))` (trailing whitespace stripped, consecutive newlines collapsed to two); for all nodes, `anchor_checksum = sha256(preceding_3_lines + node_body + following_3_lines)`.
3. Cross-artifact boundary: `content_hash` is the `expected_content_hash` contract consumed by SPEC-03 patch engine at pre-apply gate time (forward dep). `anchor_checksum` is the surrounding-drift detector consumed by the same engine. Both flow through the SQLite `nodes` and `anchor_checksums` columns — this ticket does NOT write those rows (SPEC-01-007 does) but establishes the byte-level contract.
4. Cross-platform stability: `sha256` from `node:crypto` is cross-platform stable (no endian or locale sensitivity). `yaml` package's `stringify` with `sortKeys: true` option produces deterministic output. `String.prototype.normalize('NFC')` is applied to all text before hashing to guard against future reviewers pasting NFD-encoded content.
5. FOUNDATIONS principle under audit: **Rule 6 (No Silent Retcons)** — anchor checksums detect the "surrounding prose changed but the CF YAML didn't" drift that would otherwise be a silent retcon of attribution context.

## Architecture Check

1. Splitting Pass 1 (AST production) from Pass 4 (hashing) is cleaner than fusing because Passes 2 and 3 need the mdast without the hash overhead on each re-visit. Hashing happens once, at the end. No alternatives considered that would improve this split.
2. Putting `sha256Hex(input: string | Buffer): string` in `src/hash/content.ts` as a single-purpose utility (and never exposing it from the package root) mirrors SPEC-01-001's no-library-surface invariant. No backwards-compatibility shims.
3. Keeping `remark-gfm` in the default processor chain (rather than conditionally enabling it per file) means every domain file parses under the same grammar — GFM-table cells with attribution comments parse uniformly whether the file uses tables or not. Tests cover both table-carrying and plain-prose files.

## Verification Layers

1. Running the full `build animalia` pipeline twice produces identical `content_hash` values across every node → determinism integration test (deferred to SPEC-01-009; this ticket's unit tests cover the hash primitives).
2. Editing a byte in a node's body changes `content_hash` → unit test on `canonicalHash(nodeBody)` with before/after strings.
3. Editing a byte in the 3-line surrounding context but NOT in the body changes `anchor_checksum` but NOT `content_hash` → unit test pair proving the two hashes are independent drift detectors.
4. A round-trip of `serializeStableYaml(parse(yamlText))` produces text whose sha256 matches the corresponding `content_hash` from `build` → Pass 2/Pass 4 cross-check (partial scope here; full check in SPEC-01-005).
5. FOUNDATIONS alignment: `docs/FOUNDATIONS.md` §Rule 6 (No Silent Retcons) is preserved by anchor_checksum detection → cited in module header comment.

## What to Change

### 1. `tools/world-index/src/parse/markdown.ts`

Exports `parseMarkdown(source: string): { tree: Root; lines: string[] }`:

- Constructs the unified processor: `unified().use(remarkParse).use(remarkGfm)`.
- Runs the processor on `source` (UTF-8 text), returns the mdast `Root` with `position` fields populated by remark.
- Also returns `lines: string[]` (the source split on `/\r?\n/`) — needed by Pass 4's anchor-checksum surrounding-lines lookup. Computing once at Pass 1 time avoids re-splitting in Pass 4.
- No `remark-frontmatter` — world files don't carry top-of-file YAML frontmatter at the domain-file level; fenced YAML blocks inside `CANON_LEDGER.md` are parsed as `code` nodes with `lang='yaml'` per SPEC-01 Pass 2.

### 2. `tools/world-index/src/hash/content.ts`

Exports:
- `sha256Hex(input: string): string` — `crypto.createHash('sha256').update(input.normalize('NFC'), 'utf8').digest('hex')`.
- `normalizeProseWhitespace(body: string): string` — strips trailing whitespace per line, collapses consecutive `\n` runs to at most `\n\n`, strips leading and trailing blank lines.
- `serializeStableYaml(value: unknown): string` — wraps `yaml.stringify(value, { sortMapEntries: true, lineWidth: 0 })`. Spec requires keys sorted + whitespace normalized.

### 3. `tools/world-index/src/parse/canonical.ts`

Exports:
- `contentHashForYaml(parsed: unknown): string` — returns `sha256Hex(serializeStableYaml(parsed))`.
- `contentHashForProse(body: string): string` — returns `sha256Hex(normalizeProseWhitespace(body))`.
- `anchorChecksum(lines: string[], lineStart: number, lineEnd: number): string` — concatenates the 3 preceding lines (empty string if the node starts near file start), the node body (`lines.slice(lineStart - 1, lineEnd).join('\n')`), and the 3 following lines, then returns `sha256Hex(concat)`. Line indices are 1-based to match remark's `position.start.line` convention.

### 4. Unit-test fixtures

Add minimal `tools/world-index/tests/fixtures/` samples:
- `fixture-prose.md` — a small markdown file with a section containing a paragraph + a GFM table with an attribution comment in a cell.
- `fixture-ledger.md` — a small file with one fenced-YAML CF record and one fenced-YAML CH record.

Tests compute hashes on these fixtures and compare against hard-coded expected values (baseline fixed at first successful run; future drift flags a bug).

## Acceptance Criteria

### Tests That Must Pass

1. `parseMarkdown(fixture-prose.md)` returns a `Root` whose children include expected `heading`, `paragraph`, `table`, and `html` (for the attribution comment) nodes with non-null `position` fields.
2. `contentHashForProse("  foo  \n\n\n\nbar\n")` equals `contentHashForProse("foo\n\nbar")` (normalizeProseWhitespace idempotence).
3. `contentHashForYaml({b: 2, a: 1})` equals `contentHashForYaml({a: 1, b: 2})` (key-order independence).
4. `anchorChecksum` of a node changes when a byte in the 3-line surrounding context changes, and is unchanged when a byte 10+ lines away changes.
5. Re-running `parseMarkdown` + `contentHashForProse` on the same source twice in the same process produces identical hashes (determinism within a process).
6. Same, across two fresh Node processes (determinism across processes).

### Invariants

1. Every hashing entry point routes through `sha256Hex`, which applies NFC normalization — no function in `parse/` or `hash/` computes a hash without the normalization step (grep-proof: `crypto.createHash` only appears in `hash/content.ts`).
2. `serializeStableYaml` is the only YAML-to-string path used for canonical hashing — no parser or CRUD module calls `yaml.stringify` directly (grep-proof).
3. `anchorChecksum` depends on exactly 3 surrounding lines on each side — magic numbers centralized as `ANCHOR_CONTEXT_LINES = 3` so future adjustments land in one place.
