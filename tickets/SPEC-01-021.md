# SPEC-01-021: Retune `named_entity` extraction against proposal workflow labels and phrase-fragment captures

**Status**: PENDING
**Priority**: HIGH
**Effort**: Medium
**Engine Changes**: Yes — `tools/world-index` heuristic entity extraction, source-node eligibility, and live `animalia` regression coverage.
**Deps**: `archive/tickets/SPEC-01-020.md`, `archive/tickets/SPEC-01-019.md`, `docs/FOUNDATIONS.md`, `archive/specs/SPEC-01-world-index.md`

## Problem

Fresh rebuild audit on 2026-04-22 showed `worlds/animalia/_index/world.db` is structurally clean (`verify animalia` succeeds, `validation_results = 0`, unresolved edges = `0`, dangling edge/mention references = `0`), but the semantic `named_entity` surface still contains reproducible false positives after rebuild.

The rebuilt DB contains `224` `named_entity` rows, `218` with `Kind: unknown`. Reproduced noise falls into two bounded classes:

1. Proposal workflow labels emitted from whole-file `proposal_card` nodes, including:
   - `Distribution Discipline` (`12` mentions across `12` proposal files)
   - `Repairs Applied` (`12` mentions across `12` proposal files)
   - `World Laundering` (`8` mentions across `8` proposal files)
2. Heading-like or sentence-fragment captures emitted from broad prose nodes, including:
   - `Breakage Points` (`52` mentions across `12` files)
   - `Regional Asymmetry` (`26` mentions across `12` files)
   - `Value Stores` (`21` mentions across `7` files)
   - `In Brinewick` (`14` mentions across `5` files)
   - `An Ash` (`7` mentions across `3` files)

These are not stale checked-in artifact residue; they reproduce immediately after `npm run build`, `node tools/world-index/dist/src/cli.js build animalia`, and `node tools/world-index/dist/src/cli.js verify animalia`.

Live whole-file record behavior: `tools/world-index/src/parse/prose.ts` creates `proposal_card` nodes from the entire file body (`body: lines.join("\n")`), so proposal workflow headings such as `Phase 6c (Distribution Discipline)`, `Phase 6d (Diegetic-to-World Laundering)`, and `Phase 6f Repairs Applied` are eligible for heuristic entity extraction.
3. Live heuristic behavior: `tools/world-index/src/parse/entities.ts` still scans those persisted prose-node bodies with `CAPITALIZED_MULTIWORD_REGEX`, while `tools/world-index/src/parse/stoplist.ts` only blocks the earlier audited labels from `archive/tickets/SPEC-01-018.md` and `archive/tickets/SPEC-01-020.md`.
4. Rebuilt DB evidence confirms the remaining seam is semantic rather than structural: `entity_mentions` join cleanly to source nodes and entity rows, but proposal workflow labels and phrase fragments still persist as first-class `named_entity` rows with nontrivial mention counts.

## Architecture Check

1. The clean fix is to narrow heuristic eligibility and/or candidate shaping at the extractor boundary so proposal workflow headings and phrase fragments never become persisted entities, rather than teaching downstream consumers to maintain an ever-growing semantic blacklist.
2. No backwards-compatibility aliasing or parallel "filtered entity" layer should be introduced. The canonical `named_entity` surface itself should become trustworthy.

## Verification Layers

1. Proposal workflow labels no longer persist as `named_entity` rows after a fresh `animalia` rebuild. -> readonly DB query for `Distribution Discipline`, `Repairs Applied`, and `World Laundering`
2. Phrase-fragment captures no longer persist as `named_entity` rows after a fresh `animalia` rebuild. -> readonly DB query for `In Brinewick`, `An Ash`, and agreed heading-furniture examples
3. Legitimate world entities remain present after the retune. -> readonly DB query for retained positives such as `Vespera Nightwhisper`, `Melissa Threadscar`, `Copper Weir`, and `Ash-Seal commercial-company Brinewick anomaly`
4. The rebuilt DB remains structurally clean after the semantic retune. -> `verify animalia` plus readonly unresolved/dangling queries

## What to Change

### 1. Reassess proposal-card eligibility for heuristic entity extraction

Audit whether whole-file `proposal_card` and related proposal batch/card node types should feed heuristic entity extraction directly. If they remain eligible, strip workflow-phase headings and other process scaffolding before heuristic scanning.

### 2. Retune phrase capture away from heading furniture and sentence fragments

Tighten `tools/world-index/src/parse/entities.ts` so repeated heading labels and article/preposition-led fragments like `In Brinewick` and `An Ash` stop becoming first-class entities.

### 3. Add live regression coverage at the rebuilt-`animalia` seam

Encode the audited false positives directly in package tests so future rebuilds cannot silently repollute the semantic layer.
